# MCSManager Network Notes

MCSManager 的 Web 和 Daemon 是两个服务。Web 面板负责界面，Daemon 负责实例管理、文件传输、日志和控制台。

关键点：浏览器也需要能直接访问 Daemon。只让 Web 能访问 Daemon 不够。

`127.0.0.1` 不是一个通用节点地址。它取决于“谁在访问”：

- NUC 宿主机浏览器访问 `127.0.0.1:24444`：会命中宿主机映射出来的 Daemon 端口。
- Web 容器内部访问 `127.0.0.1:24444`：访问的是 Web 容器自己，不是宿主机，也不是 Daemon。
- 远程电脑浏览器访问 `127.0.0.1:24444`：访问的是远程电脑自己，不是 NUC。

因此节点地址建议使用一个“浏览器和 Web 后台都能访问”的地址：NUC 局域网 IP，或 Sakura Frp 的 Daemon 隧道公网地址。

## 节点地址怎么填

### 推荐：局域网访问

`.env` 默认已经是局域网模式：

```dotenv
MCSM_WEB_BIND=0.0.0.0
MCSM_DAEMON_BIND=0.0.0.0
```

节点里填写：

```text
地址: NUC 的局域网 IP
端口: 24444
```

建议在路由器里给 NUC 做 DHCP 静态绑定，避免重启或换线后 IP 变化。

### 可选：Sakura Frp 远程访问

需要至少两条隧道：

```text
Web    -> 127.0.0.1:23333
Daemon -> 127.0.0.1:24444
```

节点里填写 Daemon 那条隧道的公网地址和远程端口，不要填 `127.0.0.1`。

### 仅限测试：NUC 本机浏览器

```text
地址: 127.0.0.1
端口: 24444
```

这个只保证 NUC 本机浏览器可以直连 Daemon，不保证 Web 容器后台的连通性检测也能通过。

## 密钥

启动 Daemon 后在 NUC 上执行：

```bash
./scripts/mcsm.sh key
```

它会打印 Daemon 配置文件路径、可能的密钥字段和节点地址填写建议。

## 常见错误

- Web 能打开，但节点显示网络连接失败：通常是 Daemon 地址对 Web 后台或当前浏览器不可达。
- 网页直连失败：浏览器访问不到节点里填写的 Daemon 地址。
- 节点里填 `127.0.0.1`：只有在 NUC 本机浏览器测试时才可能成立；远程浏览器会连到当前电脑自己，Web 容器会连到容器自己。
